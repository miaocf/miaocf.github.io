网页发布

  1. hexo g

    Generator 指令， 该指令可以快速将 hexo 根目录下 source 资源文件夹中的图片、markdown 文档内容转换成 index.html 文件， 并结合当前设置的主体，渲染成最终的静态网页文件，存放于public 文件夹下。

    选项 描述
    -d, --deploy 文件生成后立即部署网站
    -w, --watch 监视文件变动
    -b, --bail 生成过程中如果发生任何未处理的异常则抛出异常
    -f, --force 强制重新生成文件 Hexo 引入了差分机制，如果 public 目录存在，那么 hexo g 只会重新生成改动的文件。 使用该参数的效果接近 hexo clean && hexo generate
    -c, --concurrency 最大同时生成文件的数量，默认无限制

  2. hexo server

    该指令启动服务器，可以让我们在本地查看网页效果。默认端口为：

    http://localhost:4000/

    选项 描述
    -p, --port 重设端口
    -s, --static 只使用静态文件
    -l, --log 启动日记记录，使用覆盖记录格式

  3. hexo deploy

    部署网址指令，改指令将public 文件夹下的静态网页文件部署到 github 网站上，在本地即体现为将publick 文件夹的内容复制到了.deploy_git 文件夹下。
