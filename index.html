nRF52840开发（四）

发表于 2022-03-24

简介

学习nRF52850 MCU 的寄存器类型以及区别，了解MCU和外设之间的资源共享机制。掌握哪些外设可以同时使用，哪些外设只能同时使用一个。

寄存器类型

nRF52840的寄存器可以分为以下四种类型：

  • 任务寄存器 Task： 可以有程序或者时间出发，用于启动某一个任务，比如用 UART 发送数据时。

    具体使用时，直接向任务寄存器的地址写 “1” 即可触发对应的任务。

    eg: UART 的 STARTTX 任务，对应的任务寄存器地址为 0x400020000， 所以应用程序只需要向这个地址写 “1” 即可触发 UART 的 STARTTX 任务。

  • 事件寄存器 Event: 外设产生事件时，外设通过对对应的事件寄存器和中断向 CPU 提供指示。

  • 快捷方式寄存器： 快捷方式寄存器用于提供一些快捷的操作而不用去配置多个寄存器。

    nRF52840 中连接同一个外设的事件和任务，当任务发生时， 自动触发任务而不需要 CPU 参与。

  • 通用寄存器： 和一般单片机的寄存器一样。

任务和事件的一个很重要的用处是和 PPI （外设交叉互联）配合使用，通过配置 PPI， 将某个事件和任务连接起来，连接后，当事件产生时会在硬件上自动触发任务而不需要 CPU 参与。这种机制可以有效的减少 CPU 的占用时间，降低 CPU 的负荷。 同时也节省了中断资源，提高事件的响应速度。

外设资源共享

nRF52840 的每一个外设都分配了固定大小位 0x1000 字节的地址空间，每个外设都有一个ID， ID和外设的对应关系是： 从起始地址为 0x400000000 的外设开始编号，即起始地址为 0x400000000 的外设ID为0， 起始地址为 0x40001000 的外设 ID 是1， 以此类推。

具有相同 ID 的外设共享寄存器等资源，所以具有相同 ID 的外设同时只能使用一个 。

控制原理

控制原理主要分为两类： 高电平有效原理和低电平有效原理

通常我们采用低电平有效原理：

  1. 单片机 IO 低电平时灌入电流一般比高电平时的拉电流要打，能提供足够的电流驱动LED
  2. 单片机上电或者复位的时候，IO 口一般都是高阻输入，可以保证此时属于熄灭状态。

限阻电流

LED 限阻电流由下面公式计算：
$$
R = \frac{(Vcc - V_F)}{I} \Omega
$$
典型的Vcc = 3.3 V， 电流值I = 20 mA, 正向压降 VF = 2.0 V， 即计算得到R = 650 Ω。在52840上面的阻值为1 k Ω。这里电流可以根据实际亮度需求来取值。

1. 简介

C语言中，数组元素访问可以使用指针或者下标两种方法。二者效率上在某些情况下有所不同。

在阅读《C和指针》第八章第145页时，有所感悟。

前置知识：

  • 数组名和指针的区别

    1. 数组具有确定数量的元素，而指针只是一个标量值。

    2. 数组名是一个指针常量，而非指针变量。

  • 数组名和指针不能替换的场景：

    1. 作为sizieof操作符时，数组返回元素个数，指针返回指定环境下的指针大小。（32位为4，64位为8）
    2. 作为操作符 & 的操作数时，取一个数组名的地址返回一个指向数组的指针。而不是一个指向某个指针常量的指针。

Android系统架构

发表于 2021-09-24 更新于 2022-02-24 分类于 Android

简介

安卓系统整体采用了分层的架构，总共可以分为四层。从高层到低层分别是Android应用层，Android框架层，Android系统运行库层和Linux内核层。整个系统框架图如下图所示。

Android系统框架主要应用于ARM平台，但不局限于ARM平台，通过编译器控制在X86、MAC等架构的机器上也可以运行。

Linux 内核层

安卓平台的基础是2.6版本的Linux内核，内核层主要提供一些底层功能，例如安全、内存管理、进程管理、网络堆栈、驱动模型，谷歌允许设备制造商为Linux内核开发硬件驱动程序。

Google在Android架构里提出了Hal的概念，这个HAL其实就是独立的意思，Android系统不仅依赖于某一个具体的硬件驱动，而是依赖于Hal代码，这样，第三方厂商可以將自己不开源的代码封装在HAL层，仅仅提供二进制代码。从而避免了GPL协议的污染。

HAL层

硬件抽象层（Hardware Abstraction Layer, HAL)，简称为HAL层。 HAL层通过更高级别的Java语言API框架显示设备硬件功能。包含扩个库模块，其中每个模块都为特定类型的硬件组成实现一个界面。例如相机或者蓝牙模块。 当框架API 要求访问硬件时，安卓系统将会加载对应的库模块。HAL层处于系统运行库层，链接Linux层底层硬件驱动和Framework层。

而Android采用了Apache Licence2.0协议发布，Apache Licence2.0协议并未要求使用基于Apache Licence2.0协议的源代码的软件也必须开源。由于HAL属于Android的一部分，自然也不必开源了。当然加入HAL还有其他目的：统一硬件的调用接口；针对一些特殊的要求。

https://www.cnblogs.com/ww3344/p/5616983.html

HAL 是为了链接driver与FRAMEWORK层的很好的东西,方便与app层开发人员,你无须知道driver层的信息,就可以做相关开发。

driver要有，但是有一部分涉及核心技术的使用hal来保护自己的核心技术不被泄露。换句话就是kernel中的driver调用hal中实现的方法来驱动，这样hal中的源码就不会被暴露出来，只暴露一个接口给kernel调用。kernel中虽然存在着driver，但是具体的实现方法没有，需要调用hal中的方法来具体实现。

HAL层是为了硬件厂商规避GPL协议而存在的 所以不是必要的条件没有也可以执行

然后这个HAL层是为了实现硬件的描述 进行了抽象 ， 对上层提供接口 ，与linux kernel驱动进行通信 数据的读写 。有关硬件实现的细节，本来我们是要在linuxkernel层的驱动实现的 ，这样我们就可以通过HAL层来把硬件实现，既规避了GPL 又能提供接口 。

运行时库层

Android Runtime

Android 5.0 以上版本的系统，每个应用都在自己的进程中运行，并且有自己的Android Runtime (ART)实例。 ART通过执行DEX格式的文件在低内存设备上运行多个虚拟机。

原生C、C++ 库

许多核心Android 系统组件和服务的构建依赖C 、 C++ 编写的原生库。 Android通过提供Java框架API 以向应用显示其中部分原生库的功能。

Framework层

Framework层主要是由 Java 语言进行编写的API层，同时也是开发者开发过程中大部分涉及到的层。通过 API 接口，开发者可以方便的调用Android系统的整个功能集。Framework层可以简化核心模块系统组件和服务的重复使用。

Framework层包括活动管理器、窗口管理器、内容提供者、视图系统、包管理器、电话管理器、资源管理器、位置管理器、通知管理器和XMPP服务十个部分。

Applications层

Application 层即日常用户最经常接触到的一层架构。用户手机上的一App应用以及Android OS自带的一些原生应用都是存在于应用层。典型的应用有联系人、邮件、日历、地图和短信等。开发者通过编写Java / Kotlin语言的应用程序， 与原生系统程序和平共处。

正则表达式

发表于 2021-07-31 更新于 2022-02-24 分类于 Shell

一、基本概念

正则表达式（regular expression)是一种字符串的匹配模式，可以用来检测一个字符串是否含有按照某种句法规则的字符串子串。在许多文本编辑器中，正则表达式常常被用来检索和替换某个模式的文本内容。通常被缩写为**”regex”**。

二、 表达式

1. 通配符

  • 常用的通配符有“ * ”、“ ？”

    其中“ * ”表示匹配0个或者任意多个字符；” ? “表示匹配0个或者1个字符。

    • ^ b表示匹配输入字符串的开始位置。
    • [0-9]+ 匹配多个数字， [0-9] 匹配单个数字，+匹配一个或者多个。
    • abc$ 匹配字母abc并以abc为结尾， $ 为匹配输入字符串的结束位置。

  • 例注册表注册只允许以字母数字下划线和连接符(-)， 就可以通过以下正则表达式来表示。
